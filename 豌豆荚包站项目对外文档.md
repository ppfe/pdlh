# 豌豆荚包站项目外部接入方案 #

## 页面接入逻辑 ##
> 通过 `Wdlh` Js方案，接入方不用考虑豌豆荚内部下载的实现逻辑，只需要简单的几步就可以轻松完成接入

1. 页面 js 调用 ```Wdlh.init``` 初始化 sdk
2. 页面 js 调用 ```Wdlh.checkExist``` 检测包名是否存在
3. 用户点击【普通下载】,页面js调用 `Wdlh.downloadNormal` 触发普通下载逻辑
4. 用户点击【高速下载】,页面js调用 `Wdlh.downloadFast` 触发高速下载逻辑
	- 已安装豌豆荚时，唤起豌豆荚进行高速下载和安装
	- 未安装豌豆荚，进行豌豆荚下载，安装并启动豌豆荚即可完成对目标应用的高速下载和安装

## 1. 申请接入 ##
业务方接入，向豌豆荚接口人申请分配接入所需要的豌豆荚渠道 `channel`

## 2. 开始接入 ##

豌豆荚为第三方接入提供了方案相关的JS库，直接引用即可

	<script src="https://cdnjs.cloudflare.com/ajax/libs/zepto/1.2.0/zepto.min.js"></script>
	<script src="https://img.ucdl.pp.uc.cn/upload_files/js/wdj-dl-highspeed-sec.js"></script>

> 如果页面已有jQuery或者zepto(^1.2.0)，则不需要再引用 `zepto.min.js`，注意zepto要1.2.0以上，不然会不支持android 2.3.x以下版本

## 3. 接口文档 ##
引用本JS后，会自动在window对象挂载接口对象 `Wdlh`

### 一. 初始化接口 `Wdlh.init` ###
该接口全局只需要调用一次，主要用于配置一些全局信息比如 `packageName`、`chSrc` 等。页面 js 在使用其他接口前必须确保已经调用过该接口。
```js
Wdlh.init(config);
```
#### 1.1 config 为对象类型，可传字段如下： ####
| 配置项     | 说明                             | 数据类型    | 缺省值   |
| ----- | ------------------------------- | ------- | ----- |
| packageName     | (必填) 应用包名 | String  | 无  |
| channel   | (必填) 渠道号标识，用于指定要安装的豌豆荚渠道包，由豌豆荚官方提供 | String | 无 |
| ch        | (选填) 来源栏目/二级渠道 | String | default |

#### 示例 ####
```js
 Wdlh.init({
	packageName: 'xxx',
	channel: 'yyy',
	ch: 'zzz'
});
```

### 二、包名检测接口 `Wdlh.checkExist` ###
```js
Wdlh.checkExist(callback, config);
```
#### 1.1 `callback` 为回调函数，返回一个对象，里面包含结果详情信息 ####
返回对象里包含以下字段

| 字段     | 说明                             | 数据类型 |
| -----     | ------------------------------- | ------- |
| code   | 0: 成功，1: 参数出错, 2: 资源库查无此应用, 3: 网络请求失败 | Number  |
| data   | 对象类型，包含以下字段</br> {String} downloadUrl - 该包对应的下载链接，只在请求成功时返回</br> {String} msg - 接口运行结果描述 | Object |

#### 1.2 config 为对象类型，可传字段如下： ####

| 配置项     | 说明                             | 数据类型    | 缺省值   |
| -----     | ------------------------------- | ------- | ----- |
| packageName     | (选填) 应用包名，当要检测的包名和 `init` 接口传入的包名不一致时才需要传入 | String  | 无  |
| ch        | (选填) 来源栏目/二级渠道 | String | default |

#### 示例 ####
```js
// 检测全局配置的包是否存在
Wdlh.checkExist(function(ret) {
	if (!ret.code) {
		console.log('Wdlh.checkExist ok.');
	} else {
		console.log('Wdlh.checkExist failed.');
	}
});

// 检测自定义配置的包是否存在
Wdlh.checkExist(function(ret) {
	if (!ret.code) {
		console.log('Wdlh.checkExist ok.');
	} else {
		console.log('Wdlh.checkExist failed.');
	}
}, {
	packageName: 'xxx',
	ch: 'zzz'
});
```

### 三、普通下载接口 `Wdlh.downloadNormal` ###
用户进行普通下载行为时调用的方法，接口请求成功时会返回一个下载链接，接入方再自行处理下载逻辑。
```js
Wdlh.downloadNormal(callback, config);
```
#### 1.1 `callback` 为回调函数，返回一个对象，里面包含结果详情信息 ####
返回对象里包含以下字段

| 字段     | 说明                             | 数据类型    |
| -----     | ------------------------------- | ------- |
| code     | 0: 成功，1: 参数出错, 2: 资源库查无此应用, 3: 网络请求失败 | Number  |
| data   | 对象类型，包含以下字段</br> {String} downloadUrl - 该包对应的下载链接，只在请求成功时返回</br> {String} msg - 接口运行结果描述 | Object |

#### 1.2 config 为对象类型，可传字段如下： ####

| 配置项     | 说明                             | 数据类型    | 缺省值   |
| -----     | ------------------------------- | ------- | ----- |
| packageName     | (选填) 应用包名，当要检测的包名和 `init` 接口传入的包名不一致时才需要传入 | String  | 无  |
| ch        | (选填) 来源栏目/二级渠道 | String | default |
#### 示例 ####
```js
// 下载全局配置的包
Wdlh.downloadNormal(function(ret) {
	if (!ret.code) {
		console.log('Wdlh.downloadNormal ok.');
		window.location.href = ret.downloadUrl;
	} else {
		console.log('Wdlh.downloadNormal failed.');
	}
});

// 下载自定义配置的包
Wdlh.downloadNormal(function(ret) {
	if (!ret.code) {
		console.log('Wdlh.downloadNormal ok.');
		window.location.href = ret.downloadUrl;
	} else {
		console.log('Wdlh.downloadNormal failed.');
	}
}, {
	packageName: 'xxx',
	ch: 'zzz'
});
```

### 四、高速下载接口 `Wdlh.downloadFast` ###
用户进行普通下载行为时调用的方法，接口请求成功时会返回一个下载链接，接入方再自行处理下载逻辑。
```js
Wdlh.downloadFast(callback, config);
```
#### 1.1 `callback` 为回调函数，返回一个对象，里面包含结果详情信息 ####
返回对象里包含以下字段

| 字段     | 说明                             | 数据类型    |
| ----- | ------------------------------- | ------- |
| code     | 0: 成功，1: 参数出错, 2: 资源库查无此应用, 3: 网络请求失败 | Number  |
| data   | 对象类型，包含以下字段</br> {String} downloadUrl - 该包对应的下载链接，只在请求成功时返回</br> {String} msg - 接口运行结果描述</br> {Number} status - 0: 呼起豌豆荚客户端失败，直接下载豌豆荚客户端, 1: 成功呼起豌豆荚客户端进行高速下载 | Object |

#### 1.2 config 为对象类型，可传字段如下： ####
| 配置项     | 说明                             | 数据类型    | 缺省值   |
| -----     | ------------------------------- | ------- | ----- |
| packageName     | (选填) 应用包名，当要检测的包名和 `init` 接口传入的包名不一致时才需要传入 | String  | 无  |
| ch        | (选填) 来源栏目/二级渠道 | String | default |

```js
// 下载全局配置的包
Wdlh.downloadNormal(function(ret) {
	if (!ret.code) {

		console.log('Wdlh.downloadNormal failed.');
	}
});

// 下载自定义配置的包
Wdlh.downloadNormal(function(ret) {
	if (!ret.code) {
		console.log('Wdlh.downloadNormal ok.');
		window.location.href = ret.downloadUrl;
	} else {
		console.log('Wdlh.downloadNormal failed.');
	}
}, {
	packageName: 'xxx',
	ch: 'zzz'
});
```

## FAQ ##

### 1、为什么手机明明安装了豌豆荚，还是调不起豌豆荚进行高速下载？ ###
豌豆荚调起协议需要豌豆荚进程在活动状态下才可以进行，所以调起不成功，代表的是豌豆荚没安装 或者 豌豆荚已安装但没启动。
